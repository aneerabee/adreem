## ما تم تثبيته في هذه الجولة

- تحويل الويب في وضع API إلى cloud-only: لا قراءة دفتر من `localStorage`، لا حفظ دفتر محلي، ولا snapshots محلية في الرابط المنشور.
- حفظ دخول الجهاز في `localStorage` كتذكر دخول طويل الأجل، مع إبقاء بيانات الدفتر نفسها cloud-only بدون قراءة دفتر مالي محلي قديم.
- اختبار رفض API لأي web token غير معروف قبل الوصول لأي دفتر.
- إيقاف قبول web ledger tokens القديمة في API، والاعتماد على session tokens الناتجة من الإيميل وكلمة المرور فقط.
- تحديث فحص التشغيل `verify:runtime` ليسجل الدخول بـ `ADREEM_RUNTIME_TEST_EMAIL/PASSWORD` ثم يختبر `/api/ledger`.
- إضافة اختبار يثبت أن توكنات env القديمة لا تفتح الدفتر.
- إضافة اختبار يثبت أن جلستين مختلفتين تذهبان إلى دفترين منفصلين قبل الوصول إلى repository.
- تقوية API حتى لا يستطيع test repository تجاوز التحقق من token أثناء الاختبارات.
- تحديث متطلبات env على Contabo: إزالة `ADREEM_WEB_LEDGER_*` بالكامل، واستخدام `ADREEM_RUNTIME_TEST_EMAIL/PASSWORD` مع backup وaudit وattachments.
- تصحيح remote السيرفر إلى `https://github.com/aneerabee/adreem.git`.
- تحويل أداة `npm run ops:create-ledger-access` إلى توافق قديم بدون إنشاء روابط token؛ إنشاء المستخدمين يتم من صفحة الإدارة بالإيميل وكلمة المرور.
- إضافة سجل مستخدمين ديناميكي: صاحب النظام يستطيع إضافة مستخدم مستقل من صفحة إدارة الويب بدون تعديل env في كل مرة.
- المستخدم غير المضاف لا يدخل الدفتر، لكنه يرى Telegram ID الخاص به فقط ليرسله لصاحب النظام.
- صفحة الإدارة تنشئ مستخدمًا بإيميل وكلمة مرور، وتحفظ password hash فقط؛ الويب والبوت يعملان على نفس `ledgerId`.
- تحديث Contabo إلى commit `f025c06` وتفعيل `ADREEM_TELEGRAM_ADMIN_IDS` و`ADREEM_TELEGRAM_USERS_FILE`، مع restart ناجح للبوت والـ API.
