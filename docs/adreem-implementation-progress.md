## ما تم تثبيته في هذه الجولة

- عزل التخزين المحلي في الويب حسب API token بدون كتابة التوكن الخام داخل مفاتيح `localStorage`.
- اختبار رفض API لأي web token غير معروف قبل الوصول لأي دفتر.
- توسيع اختبارات توازي الويب والبوت:
  - تحويل دينار.
  - تحويل دولار.
  - مصروف.
  - بيع دولار.
  - شراء دولار.
  - حركة ناقصة.
- تفعيل نوع حركة `دخل` في الويب والبوت.
- جعل حركات الدخل/التصحيح التي لا تحتاج مصدرًا تتجاوز خطوة `من` في الويب والبوت.
- إضافة تمييز بصري أولي لحركة الدخل.
- قصر ربط `مشروع / أصل` على حركات الدخل والمصروف فقط، ومنع ظهوره في التحويل وبيع/شراء الدولار.
- توحيد فرق المطابقة في core مشترك مع `diffDinar` و `diffUsd`.
- تقوية metadata المرفقات: نوع ملف، حجم، رابط/مسار، وحد حجم.
- نقل إيقاف الحركات المتكررة إلى core بدل منطق inline في الواجهة.
- نقل بناء التنبيهات إلى core قابل للاختبار.
- إضافة فلاتر السجل: بحث، نوع حركة، حالة.
- إضافة فلترة السجل بالحساب.
- إضافة تنبيه حركة كبيرة وتنبيه تكرار محتمل من core قابل للاختبار.
- إضافة snapshot قبل ترحيل legacy وجعل النسخ المحلية تعمل حتى لو كانت قائمة النسخ القديمة فاسدة.
- نشر الويب على GitHub Pages وتحديث السيرفر إلى `cdf3ad2`.
- تحديث Command Center بعد تحقق الروابط والسيرفر.

## نتائج التحقق المؤكدة

- `npm test`: نجح، 102 اختبارًا.
- `npm run lint`: نجح.
- `npm run build`: نجح.
- فحص Playwright للهاتف والكمبيوتر: لا يوجد horizontal overflow أو clipping في الإدخال/السجل، وفلاتر السجل ظاهرة، وربط المشروع/الأصل لا يظهر في التحويل.
- فحص Playwright بعد فلتر الحساب: 4 عناصر فلترة ظاهرة، ولا يوجد overflow أو clipping على الهاتف والكمبيوتر.
- فحص السيرفر: `adreem-api.service` و `adreem-bot.service` يعملان، و`npm run verify:runtime` نجح بعد تحميل `adreem.env`.
- محاولة تشغيل وكيلين مراجعة مستقلين لم تنجح بسبب امتلاء حد الوكلاء في الجلسة؛ التعويض الحالي يكون بفحوصات مستقلة محلية وبصرية.
