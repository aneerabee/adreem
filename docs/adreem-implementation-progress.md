## ما تم تثبيته في هذه الجولة

- عزل التخزين المحلي في الويب حسب API token بدون كتابة التوكن الخام داخل مفاتيح `localStorage`.
- اختبار رفض API لأي web token غير معروف قبل الوصول لأي دفتر.
- دعم `ADREEM_WEB_LEDGER_TOKEN_HASHES` حتى لا يحتفظ السيرفر بتوكنات الويب الخام داخل خريطة الدفاتر.
- إبقاء `ADREEM_WEB_LEDGER_TOKENS` مؤقتًا للتوافق فقط، وليس كصيغة معتمدة للدفاتر الجديدة.
- تحديث فحص التشغيل `verify:runtime` ليستوعب التوكنات المشفرة، مع `ADREEM_RUNTIME_TEST_TOKEN` لاختبار `/api/ledger` من السيرفر.
- إضافة اختبار يثبت أن توكنين مختلفين يذهبان إلى دفترين منفصلين قبل الوصول إلى repository.
- تقوية API حتى لا يستطيع test repository تجاوز التحقق من token أثناء الاختبارات.
- تحديث env الحقيقي على Contabo: تحويل `ADREEM_WEB_LEDGER_TOKENS` إلى تعليق، وإضافة `ADREEM_WEB_LEDGER_TOKEN_HASHES` و`ADREEM_RUNTIME_TEST_TOKEN` مع backup.
- تصحيح remote السيرفر إلى `https://github.com/aneerabee/adreem.git`.
- إضافة أداة `npm run ops:create-ledger-access` لتوليد دفتر مستقل لكل مستخدم: `ledgerId`، رابط ويب خاص، hash للـ env، وربط Telegram اختياري.
- توسيع اختبارات توازي الويب والبوت:
  - تحويل دينار.
  - تحويل دولار.
  - مصروف.
  - بيع دولار.
  - شراء دولار.
  - حركة ناقصة.
- تفعيل نوع حركة `دخل` في الويب والبوت.
- جعل حركات الدخل/التصحيح التي لا تحتاج مصدرًا تتجاوز خطوة `من` في الويب والبوت.
- إضافة تمييز بصري أولي لحركة الدخل.
- قصر ربط `مشروع / أصل` على حركات الدخل والمصروف فقط، ومنع ظهوره في التحويل وبيع/شراء الدولار.
- توحيد فرق المطابقة في core مشترك مع `diffDinar` و `diffUsd`.
- تقوية metadata المرفقات: نوع ملف، حجم، رابط/مسار، وحد حجم.
- نقل إيقاف الحركات المتكررة إلى core بدل منطق inline في الواجهة.
- نقل بناء التنبيهات إلى core قابل للاختبار.
- إضافة فلاتر السجل: بحث، نوع حركة، حالة.
- إضافة فلترة السجل بالحساب.
- إضافة تنبيه حركة كبيرة وتنبيه تكرار محتمل من core قابل للاختبار.
